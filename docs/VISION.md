# ビジョン — medical_open_data

## なぜ作るのか

日本の医療情報は、**データとしては公開されているのに、実質的にはアクセスできない**。

厚生労働省は「医療情報ネット」のオープンデータとして全国20万件の医療施設情報をCSVで公開している。二次利用可能、無償、機械判読可能——オープンデータの要件は満たしている。

しかし現実には：

- 半年に1回のCSV配布。APIはない
- 使いたい人が自力でダウンロード→パース→DB構築→アプリ開発
- データカタログもメタデータも貧弱で、他のデータセットとの連携が困難
- 結果として、このデータを活用したサービスがほとんど存在しない

**データは公開されている。でも「使える」状態にはなっていない。**

これは医療データに限った話ではない。日本のオープンデータ全般に共通する構造的な課題だ。

## 何を作るのか

### Phase 1: 使えるAPIにする

まず、厚労省のCSVデータを**誰でもすぐに使えるREST API**として提供する。

- 全国の病院・診療所・歯科・薬局・助産所を横断検索
- 位置情報による近隣検索（「近くの病院」）
- 診療科・曜日・時間帯で絞り込み（「今やってる内科」）
- 標準的なREST API + OpenAPIドキュメント

これだけで、Webアプリ、LINEボット、チャットボット、モバイルアプリなど、何でも上に載せられる。

### Phase 2: データスペースへの接続

APIとして使える状態になったら、次は**IPA ウラノス・エコシステム・データスペース（ODS）** の枠組みに接続する。

#### ウラノス・エコシステムとは

IPAデジタルアーキテクチャ・デザインセンター（DADC）が経済産業省・NEDOと共に推進する、**組織間データ連携の国家的基盤アーキテクチャ**。

- 企業・業界・国境を横断したデータ連携
- データ主権を保ったまま流通（分散・連邦ハイブリッド型）
- 4つの疎結合レイヤ（データ / トランザクション / アイデンティティ / セマンティクス）
- OSSとして参照実装を公開中: https://github.com/Ouranos-Ecosystem-Dataspaces

参照: [ODS-RAM ホワイトペーパー](https://www.ipa.go.jp/digital/architecture/reports/ouranos-ecosystem-dataspaces-ram-white-paper.html)（2025年2月公開）

#### なぜデータスペースなのか

現在のユースケース（自動運転、インフラDX、ドローン航路、自動車LCA等）はいずれも産業領域。**医療・ヘルスケア領域のユースケースはまだない。**

しかし、医療データは最もデータスペースの恩恵を受ける領域の一つだ：

```
医療施設データ（厚労省）───┐
介護施設データ（厚労省）───┤
自治体健診データ ──────────┼── データスペース（ODS）
保険者データ ──────────────┤       │
学術研究データ ────────────┘       │
                                    ↓
                        ┌──────────────────────┐
                        │ 検索アプリ            │
                        │ 地域医療分析          │
                        │ 防災・災害時医療支援   │
                        │ 政策立案エビデンス     │
                        │ 研究・疫学            │
                        └──────────────────────┘
```

**ODS-RAMへの対応として具体的に想定すること：**

| ODSレイヤ | 本プロジェクトでの対応 |
|-----------|----------------------|
| L1 データレイヤ | データ品質管理、来歴(provenance)の記録、更新履歴の保持 |
| L2 トランザクションレイヤ | 標準APIプロトコルでのデータ提供（REST + 将来的にODSプロトコル） |
| L3 アイデンティティレイヤ | API認証・認可（公開データはオープン、詳細データは認証付き） |
| L4 セマンティクスレイヤ | 医療施設データの標準語彙・メタデータ定義（HL7 FHIR等との連携も視野） |

### Phase 3: エコシステムの形成

本プロジェクトが**「医療データ × データスペース」の実証事例**となることで：

1. **他のオープンデータ保有者**（自治体、保険者等）がデータスペースに参加する動機になる
2. **開発者コミュニティ**が標準APIの上にサービスを構築できる
3. **政策担当者**がデータ駆動の意思決定を行える

## 思想

### 壁を溶かす

このプロジェクトの根底にあるのは「**データの壁を溶かす**」という思想だ。

データは存在する。しかしCSVファイルとして厚労省のサーバーに置かれているだけでは、壁の向こうにあるのと同じだ。API化し、データスペースに接続し、誰でもアクセスできる形にすること——それが「壁を溶かす」ということだ。

### ルーテッドコスモポリタニズム

ウラノス・エコシステムは国際的な相互運用性を志向しているが、扱うデータは日本の医療制度に根ざしたものだ。**ローカルのデータに根ざしながら、グローバルな標準で開く**——Rooted Cosmopolitanismの実践でもある。

### オープンソースとして

本プロジェクトはOSSとして公開する。理由は3つ：

1. **透明性**: 公的データを扱う以上、処理過程は公開されるべき
2. **再現性**: 誰でも同じ環境を構築できる
3. **貢献**: ウラノス・エコシステムへの実証事例として

## ロードマップ

```
Phase 1 (MVP)
  ├── CSV→DB取り込みパイプライン
  ├── REST API（検索・近隣検索）
  └── SQLite / PostgreSQL / MySQL対応

Phase 2 (データスペース準備)
  ├── ODS-RAMアーキテクチャへの適合検討
  ├── メタデータ・語彙定義（セマンティクスレイヤ）
  ├── 認証・認可レイヤ
  ├── 介護サービスデータ統合
  └── データ来歴・品質管理

Phase 3 (エコシステム)
  ├── ODS準拠コネクタ実装
  ├── 他データソースとの連携実証
  ├── フロントエンド（地図UI）
  └── コミュニティ形成・ドキュメント整備
```
